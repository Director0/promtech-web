## 1. Как открыть файл config.js

1. Откройте папку с файлами сайта.
2. Найдите файл **`config.js`**.
3. Нажмите на него **правой кнопкой мыши**.
4. Выберите **«Открыть с помощью»** → **Блокнот** (или Notepad++, если установлен).

> ⚠️ **Не открывайте двойным щелчком** — браузер может запустить файл, а не открыть для редактирования.

---

## 2. Структура файла — что здесь что

Файл `config.js` состоит из двух частей:

```
COMPANY   — данные компании (название, телефон, соцсети и т.д.)
PROJECTS  — список всех проектов на сайте
```

Каждая часть выглядит так:

```javascript
const COMPANY = {
  name: "Название компании",
  ...
};

const PROJECTS = [
  { ...проект 1... },
  { ...проект 2... },
];
```

**Правило:** весь текст, который вы редактируете, всегда находится между кавычками `"..."`. Кавычки и знаки препинания вокруг **не трогайте**.

## 3. Изменить данные компании

Найдите в файле блок `const COMPANY = {` — он выглядит так:

```javascript
const COMPANY = {
  name:        "Meridian Build",
  tagline:     "Architecture & Construction",
  description: "We design and build spaces that endure...",
  telegram:    "https://t.me/your_telegram",
  whatsapp:    "https://wa.me/1234567890",
  email:       "hello@meridianbuild.com",
  phone:       "+1 (555) 000-0000",
  stats: [
    { num: "140+", label: "Projects" },
    { num: "18",   label: "Years" },
    { num: "12",   label: "Awards" },
  ]
};
```

### Что можно менять:

| Поле | Что это | Пример |
|---|---|---|
| `name` | Название компании | `"Ваша Компания"` |
| `tagline` | Короткий слоган под логотипом | `"Строительство и дизайн"` |
| `description` | Описание в шапке главной страницы | `"Мы строим..."` |
| `telegram` | Ссылка на Telegram-чат | `"https://t.me/username"` |
| `whatsapp` | Ссылка на WhatsApp | `"https://wa.me/79001234567"` |
| `email` | Электронная почта | `"info@company.ru"` |
| `phone` | Телефон | `"+7 (900) 123-45-67"` |
| `stats` | Три цифры в блоке статистики | см. ниже |

### Как изменить статистику:

```javascript
stats: [
  { num: "200+", label: "Проектов" },
  { num: "15",   label: "Лет" },
  { num: "8",    label: "Наград" },
]
```

Замените значения `num` (число) и `label` (подпись) на свои. Блоков всегда должно быть **три**.

## 4. Добавить новый проект

### Шаг 1 — Найдите список проектов

В файле найдите строку:
```javascript
const PROJECTS = [
```

Сразу после неё начинаются проекты. Каждый проект — это блок от `{` до `},`.

### Шаг 2 — Скопируйте шаблон

Возьмите готовый шаблон из [раздела 6](#6-готовый-шаблон-нового-проекта) этой инструкции и вставьте его **в начало списка**, сразу после строки `const PROJECTS = [`.

### Шаг 3 — Заполните поля проекта

Основные поля проекта:

```javascript
{
  id:        "уникальный-id-проекта",   // только латиница, без пробелов, через дефис
  name:      "Название проекта",
  category:  "Жилой",                   // тип: Жилой / Коммерческий / Культурный и т.д.
  year:      "2025",
  shortDesc: "Короткое описание — 1-2 предложения. Видно на карточке.",
  mainMedia: { type: 'image', src: 'https://ссылка-на-главное-фото.jpg' },
  thumbnail: 'https://ссылка-на-фото-для-карточки.jpg',
  sections:  [ ...разделы проекта... ]
}
```

> **Про `id`:** придумайте уникальное название латиницей, без пробелов.  
> Примеры: `"zhk-raduga"`, `"office-tsentr-2025"`, `"villa-na-rubliovke"`  
> **Два проекта с одинаковым id работать не будут.**

## 5. Виды блоков контента

Внутри каждого раздела (`sections`) проекта можно использовать несколько видов блоков. Блоки идут один за другим внутри `content: [ ... ]`.

---

### 📝 Текстовый абзац

```javascript
{ type: 'text', value: 'Ваш текст абзаца идёт сюда.' }
```

---

### 🔤 Подзаголовок внутри раздела

```javascript
{ type: 'subheading', value: 'Название подраздела' }
```

---

### 🖼 Одиночное фото

```javascript
{ type: 'image', src: 'https://ссылка-на-фото.jpg', caption: 'Подпись к фото' }
```

Дополнительно можно указать пропорции через `ratio`:

```javascript
{ type: 'image', src: 'https://...jpg', caption: 'Подпись', ratio: 'portrait' }
```

| Значение `ratio` | Пропорции | Когда использовать |
|---|---|---|
| *(не указывать)* | 16:9 — горизонтальное | Фасады, общие виды |
| `'portrait'` | 3:4 — вертикальное | Интерьеры, башни |
| `'square'` | 1:1 — квадрат | Детали, фрагменты |

---

### 🖼🖼 Два фото рядом

```javascript
{
  type: 'image-grid',
  images: [
    { src: 'https://фото-слева.jpg',  caption: 'Подпись слева' },
    { src: 'https://фото-справа.jpg', caption: 'Подпись справа' },
  ],
  caption: 'Общая подпись под двумя фото'
}
```

---

### 💬 Цитата / выделенная фраза

```javascript
{ type: 'quote', value: 'Текст цитаты или ключевой фразы проекта.' }
```

Отображается как тёмный блок с золотой полосой слева — привлекает внимание.

---

### 📊 Характеристики проекта (таблица параметров)

```javascript
{
  type: 'specs',
  items: [
    { label: 'Площадь',    value: '4 200 м²' },
    { label: 'Адрес',      value: 'Москва, ул. Примерная' },
    { label: 'Статус',     value: 'Завершён' },
    { label: 'Год',        value: '2025' },
  ]
}
```

Рекомендуется ставить **первым блоком** в разделе «Обзор проекта».

---

### 🎬 Видео вместо главного фото

Если хотите, чтобы вместо главного фото на странице проекта играло видео, замените поле `mainMedia`:

```javascript
mainMedia: { type: 'video', src: 'https://ссылка-на-видео.mp4', poster: 'https://превью.jpg' }
```

`poster` — это фото-превью, которое показывается до начала загрузки видео. Можно не указывать.

## 6. Готовый шаблон нового проекта

Скопируйте этот блок целиком и вставьте сразу после `const PROJECTS = [`. Затем заполните поля своими данными.

```javascript
  {
    id:        "vash-id-proekta",
    name:      "Название проекта",
    category:  "Жилой",
    year:      "2025",
    shortDesc: "Краткое описание проекта — 1–2 предложения для карточки на главной.",
    mainMedia: { type: 'image', src: 'https://ссылка-на-главное-фото.jpg' },
    thumbnail: 'https://ссылка-на-фото-для-карточки.jpg',
    sections: [
      {
        id:    "obzor",
        title: "Обзор проекта",
        content: [
          { type: 'text', value: 'Вводный абзац о проекте.' },
          {
            type: 'specs',
            items: [
              { label: 'Площадь',    value: '000 м²' },
              { label: 'Адрес',      value: 'Город, улица' },
              { label: 'Статус',     value: 'Завершён' },
              { label: 'Год',        value: '2025' },
            ]
          },
          { type: 'image', src: 'https://фото-1.jpg', caption: 'Подпись к фото' },
        ]
      },
      {
        id:    "kontseptsiya",
        title: "Концепция",
        content: [
          { type: 'text', value: 'Описание концепции и идеи проекта.' },
          { type: 'quote', value: 'Ключевая фраза или цитата о проекте.' },
          {
            type: 'image-grid',
            images: [
              { src: 'https://фото-2.jpg', caption: 'Подпись слева' },
              { src: 'https://фото-3.jpg', caption: 'Подпись справа' },
            ],
            caption: 'Общая подпись'
          },
        ]
      },
      {
        id:    "rezultat",
        title: "Результат",
        content: [
          { type: 'text', value: 'Итоги и результаты проекта.' },
          { type: 'image', src: 'https://фото-финал.jpg', caption: 'Финальный вид' },
        ]
      }
    ]
  },
```

> **Разделов (`sections`) может быть любое количество.** Удалите лишние или добавьте новые по тому же принципу.

## 7. Частые ошибки и как их избежать

### ❌ Ошибка: пропущена запятая

**Неправильно:**
```javascript
{ type: 'text', value: 'Первый абзац.' }
{ type: 'text', value: 'Второй абзац.' }
```

**Правильно** — после каждого блока, кроме последнего, должна быть запятая:
```javascript
{ type: 'text', value: 'Первый абзац.' },
{ type: 'text', value: 'Второй абзац.' }
```

---

### ❌ Ошибка: апостроф или кавычка внутри текста

Если в тексте нужна кавычка `"` или апостроф `'`, используйте обратный слеш перед ним:

**Неправильно:**
```javascript
{ type: 'text', value: 'Здание «Арт-центра» открылось в '24 году.' }
```

**Правильно:**
```javascript
{ type: 'text', value: 'Здание «Арт-центра» открылось в \'24 году.' }
```

> **Подсказка:** лучше просто не использовать одиночные кавычки `'` в тексте — замените их на «ёлочки» `«»` или тире `—`. Так надёжнее.

---

### ❌ Ошибка: одинаковые `id` у проектов

Каждый проект должен иметь **уникальный** `id`. Если два проекта имеют одинаковый `id`, при переходе на один из них сайт может открыть не тот проект.

---

### ❌ Ошибка: ссылка на фото не работает

Ссылка на фото должна начинаться с `https://`. Проверьте, что она открывается в браузере — вставьте ссылку в адресную строку и убедитесь, что фото отображается.

---

### ✅ Золотое правило

**После любого изменения сохраните файл** (`Ctrl + S`) и обновите страницу в браузере (`F5`). Если что-то сломалось — нажмите `Ctrl + Z` несколько раз, чтобы отменить последние изменения.

---

## 8. Проверка результата

1. Сохраните файл `config.js` (`Ctrl + S`).
2. Откройте файл `index.html` в браузере двойным щелчком.
3. Убедитесь, что новый проект появился на главной странице.
4. Нажмите на карточку проекта — проверьте, что все разделы, фото и тексты отображаются правильно.
5. Проверьте оглавление слева (или сверху на телефоне) — все разделы должны быть в списке.

> Если страница выглядит пустой или карточки не появились — скорее всего, в файле есть синтаксическая ошибка (пропущена кавычка, запятая или скобка). Проверьте последнее место, которое вы редактировали.
